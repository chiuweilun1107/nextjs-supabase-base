Handoff · 3 steps · ~5 minutes

From design file
to merged PR.

Three steps. The first two take a minute each. The third one writes the code while you make coffee.

Step 01

Fork the repo.

One click on GitHub. The starter is yours — branch it, rename it, force-push to a private remote. The license is MIT.

Step 02

Edit DESIGN.md.

Replace the tokens with your brand: colors, type, spacing, radius. One file is the source of truth — every component reads from it.

Step 03

Run Claude Code.

Hand it a link to your design. Claude Code will implement your design — components, pages, polish, tests — and open a PR for review.

The handoff command
~/my-app · zsh
$ npx claude "Fetch my design file and implement it. https://api.anthropic.com/v1/design/h/abc123"

What you get back.

  • PR · 1Components built to match every tokenEvery Button, Input, Card, and Layout reads from DESIGN.md tokens. No magic numbers.
  • PR · 2Page templates wired to your routesMarketing, dashboard, settings — Server Components, suspense boundaries, error states.
  • PR · 3Tests + Storybook coverageSnapshot tests for every component variant, and a deployable Storybook on Vercel.
  • PR · 4A short summary of every decisionWhat was added, what was deviated from spec and why, and a list of open questions for review.
